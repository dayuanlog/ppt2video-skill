# 配色方案

当前 skill 的配色来源优先级是：

1. 用户明确指定的自定义配色要求
2. 所选 `theme-previews` 主题目录中的真实 HTML
3. 本文件作为颜色角色提炼和安全护栏

不要把本文件当成独立风格库随意挑色。若用户选择自定义风格，也要先整理颜色角色，而不是随意堆颜色。

## 1. 先从主题 HTML 里提取颜色角色

无论什么主题，先提取这几类颜色：

- `bg`：主背景
- `fg`：主文字
- `muted`：次级文字
- `accent`：强调色
- `line`：分隔线 / 边框
- `surface`：卡片 / 面板底色

如果新 scene 不确定怎么配色，先沿用主题现成变量命名和角色分配。

## 2. 当前主题家族摘要

### `guizang-magazine-demo`

- 纸感浅底或深墨底
- 衬线标题
- 暖色强调
- 更适合内容型、编辑型、案例型叙事

### `guizang-swiss-demo`

- 高对比黑白或冷白底
- 单一高纯度强调色
- 直角、网格、理性结构
- 更适合数据、方法、系统页

### `handdrawn-whiteboard-demo`

- 接近纸白底
- 黑色手绘字
- 柔和蓝色标记色
- 更适合讲解、拆解、递进

### `launch-minimal-demo`

- 黑底白字为主
- 允许插入单张白底页
- 强调色极少，通常只高亮一个词
- 更适合结论、发布会、品牌式陈述

## 3. 生成新主题时的调色原则

### 深色主题

- 背景接近黑而不是纯黑
- 正文使用接近白色的高对比文字
- 强调色只用一种
- 次级字使用 `fg` 的低透明或浅灰变体

### 浅色主题

- 背景尽量稳定，不要偏脏
- 正文字色接近黑而不是中灰
- 强调色面积小、位置准
- 卡片底色和背景至少有轻微分层

## 4. 强调色使用规则

- 一页里最多 1 个主强调色
- 不同页可以换强调方式，但同一主题家族最好保持一致
- 强调色优先给：
  - 关键词
  - 数字
  - 标签
  - 关键路径终点

不要让强调色同时铺满标题、卡片、边框、背景。

## 5. 视频导出注意点

- 核心文字不要依赖低透明度
- 大面积半透明叠色要克制
- 深色页的次级字也要保证可读
- 白底页不要用太淡的浅灰正文

## 6. 当主题里混合黑白页

这在“发布会极简”类主题中是允许且推荐的：

- 多数黑页负责抛结论
- 单张白页负责转折、呼吸或聚焦

但混合时要保持：

- 同一字体体系
- 同一强调色体系
- 同一留白逻辑

这样黑白切换才像同一个主题，而不是两套 PPT 拼起来。

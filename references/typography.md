# 排版规则

排版优先参考所选主题目录中的真实 HTML，再用本文件做护栏。

## 1. 字体选择原则

先看主题已经在用什么字体体系，再决定是否沿用。

当前主题家族大致分为：

- `guizang-magazine-demo`：偏衬线 + 编辑感
- `guizang-swiss-demo`：无衬线 + 理性结构
- `handdrawn-whiteboard-demo`：手写字体 + 简洁中文补字
- `launch-minimal-demo`：系统无衬线 + 大字发布会感

不要脱离主题去发明完全陌生的字体组合。

## 2. 字重层级

同一页通常只需要 2-3 级层级：

- 一级：主标题
- 二级：副标题 / 补充句
- 三级：标签 / 说明 / 小数字

不要一页出现 5 种字重。

## 3. 建议字号范围

| 元素 | 建议字号 |
|---|---|
| 英雄标题 | `96-160px` |
| 普通标题 | `56-88px` |
| 副标题 | `28-40px` |
| 正文 | `20-28px` |
| 标签 | `14-18px` |
| 大数字 | `56-160px` |

竖屏 `1080 x 1440` 时，标题往往应该比网页稿更大。

## 4. 行长与换行

- 标题必须有 `max-width`
- 正文必须控制行长
- 不要依赖 `<br>` 机械断行

只有在大字标题需要刻意断句时才使用 `<br>`。

## 5. 数字排版

涉及数据、编号、时间、百分比时，优先使用：

```css
.stat {
  font-variant-numeric: tabular-nums;
  font-feature-settings: "tnum" 1;
}
```

## 6. 深浅背景下的字重调整

### 深色背景

- 标题和正文要更靠近白色
- 次级文字也要保持可读
- 字重可以略高

### 浅色背景

- 正文不要用过淡灰色
- 保持足够留白
- 不要靠轻飘灰字制造“高级感”

## 7. 当前不再依赖旧字体脚本

此前文档中的字体发现脚本路径已不作为当前工作流的一部分。

当前做法更简单：

1. 先看所选主题目录中真实 HTML 已经使用的字体
2. 能复用就复用
3. 必须换字体时，再明确引入稳定资源或单文件内嵌

不要再依赖旧的自动字体发现脚本路径。

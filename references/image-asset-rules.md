# 图片素材规则

图片是证据、界面、人物或场景素材，不是随手加的装饰。

当前工作流是：

1. 先按口播拆页
2. 再决定哪些页需要图
3. 最后根据所选主题的真实 HTML 风格安排图片

## 1. 先记录素材

推荐先编号：

```md
## 图片素材
- image-01: https://...
- image-02: D:\...
```

如果用户给了多张图，不要默认全用。

## 2. 判断图片类型

每张图先判断：

- 产品界面截图
- 长截图
- 数据图 / 报表
- 人物照
- 场景图
- 品牌图
- 证据截图

## 3. 再决定放在哪一页

图片必须服务当前页语义，例如：

- 开场页：代表性主图
- 痛点页：问题截图 / 混乱证据
- 产品页：真实界面
- 证据页：前后对照 / 结果图
- 结尾页：品牌、产品名、联系信息

如果图与当前页关系不强，可以标记“不使用”。

## 4. 写进 `structure-proposal.md`

使用图片的页面必须写进 `structure-proposal.md` 的 scene 表格，优先放在“主视觉/构图”或“设计注意”字段。不要为图片单独增加额外规划文件，除非用户明确要求兼容旧管线。

至少说明：

- 用哪张图
- 这张图证明什么
- 放在画面哪个区域
- 是主图还是辅助图
- 比例如何处理
- 是否需要滚动展示

推荐写法：

```md
| 03 | 证据页 | P05-P07 | ... | 三个关键结果 | image-02，右侧证据截图，contain 显示 | 保持截图比例，不压扁；必要时做固定窗口滚动 |
```

## 5. 图片比例规则

### 截图 / 证据图

优先：

```css
img {
  width: 100%;
  height: 100%;
  object-fit: contain;
}
```

### 氛围图 / 场景图

可以：

```css
img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

但前提是不会切掉主体。

## 6. 长截图规则

长截图不要压扁。

推荐：

- 固定窗口容器
- `overflow: hidden`
- 图片 `height: auto`
- GSAP 只做 `y` 位移

不要：

- 拉伸变形
- 容器跟着图片一起动
- 给长截图叠复杂遮罩

## 7. 单文件渲染器注意事项

如果 HTML 会被“单文件上传后直接渲染”的工具消费：

- 本地相对路径可能失效
- `localhost` 资源可能失效
- 临时目录资源可能失效

优先级建议：

1. 小图标 / 小贴纸 / 字体：内嵌 `data:` URL
2. 图片：与 HTML 同目录稳定相对路径
3. 远程图床：仅在确认渲染环境能联网时使用

不要默认认为：

```css
url("./assets/xxx.png")
```

在所有工具里都能正常解析。

## 8. 渲染安全

优先：

- 实色背景承托图片
- 简单边框
- 单层阴影
- 清晰裁切容器

避免：

- 对证据图加 blur/filter
- 在截图文字区叠半透明雾层
- 用复杂 `mix-blend-mode`
- 用图片当纯装饰填充整页却没有语义价值

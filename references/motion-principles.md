# 动效原则

本文件定义当前 scene 生成的动效护栏。

主题节奏要参考 `theme-previews` 的真实 HTML，但动效边界以本文件为准。

## 1. 动效服务理解，不服务炫技

先让观众理解，再让观众感受到节奏。

一页最常见的顺序：

1. 标签或 meta
2. 主标题
3. 补充句 / 分隔线
4. 图、卡片、结构

## 2. 一页 3-6 个主动作足够

不要让所有元素都单独表演。

优先：

- 标题入场
- 1 组结构入场
- 1 个强调动作

## 3. 同页统一时间轴

所有主动画必须放在同一个 GSAP timeline 里。

不要：

- 在 timeline 外额外跑 `gsap.to()`
- 用 `repeat: -1`
- 用 Promise / async 把动画拆散

## 4. 优先安全属性

推荐：

- `opacity`
- `x`
- `y`
- 轻微 `scale`
- 轻微 `rotation`

谨慎：

- 大角度 3D
- 多层 transform 叠加

避免：

- `filter`
- `backdrop-filter`
- 复杂 `clip-path`
- 复杂 shader / canvas 循环

## 5. 节奏要有差异，但要克制

不是每页都用完全同一种入场。

可以变化：

- 先标题后卡片
- 先结构后数字
- 上下递进
- 左右对照

但不要每页都把所有元素从同一方向飞进来。

## 6. 默认不做跨页过渡

当前输出是独立 scene HTML，不是一个整页 deck。

所以默认策略是：

- 每页内部把节奏做完整
- 场景之间用剪辑或播放器切换

不要再按旧 deck 思路设计复杂跨页 transition。
